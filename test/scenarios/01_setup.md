# 01 セットアップ

ゲーム開始前のサーバー初期化・チーム準備を検証する。

---

## TC-001: プラグイン起動時のScoreboardチーム自動作成

**前提条件**
- サーバーが起動していない（またはプラグインが未ロード）

**手順**
1. サーバーを起動する（またはプラグインをリロードする）
2. コンソールまたはゲーム内で以下を実行して確認する
   ```
   /team list
   ```

**期待結果**
- [ ] `kcdk.red` チームが存在する
- [ ] `kcdk.red.captain` チームが存在する
- [ ] `kcdk.blue` チームが存在する
- [ ] `kcdk.blue.captain` チームが存在する
- [ ] `kcdk.green` チームが存在する
- [ ] `kcdk.green.captain` チームが存在する
- [ ] `kcdk.yellow` チームが存在する
- [ ] `kcdk.yellow.captain` チームが存在する
- [ ] `kcdk.admin` チームが存在する（captainなし）
- [ ] 各チームにチームカラーが設定されている
- [ ] 各チームのFriendlyFireがオフになっている
- [ ] 各チームのCollisionRuleが `NEVER` になっている

---

## TC-002: `/kcdk setup` コマンド

**前提条件**
- サーバー起動済み・OP権限あり

**手順**
1. 意図的にScoreboardからチームを削除する
   ```
   /team remove kcdk.red
   ```
2. セットアップコマンドを実行する
   ```
   /kcdk setup
   ```
3. チームリストを確認する
   ```
   /team list
   ```

**期待結果**
- [ ] `§aセットアップが完了しました。` というメッセージが表示される
- [ ] `kcdk.red` が再作成されている
- [ ] TC-001 と同じチーム構成になっている

---

## TC-003: プレイヤーのScoreboardチームへの手動割り当て

**前提条件**
- TC-001 完了済み
- プレイヤーが1名以上ログイン済み

**手順**
1. プレイヤーを `kcdk.red` チームに追加する
   ```
   /team join kcdk.red <PlayerName>
   ```
2. プレイヤーの名前タグの色を確認する
3. 同チームのプレイヤーが当たり判定で当たらないか確認する

**期待結果**
- [ ] プレイヤーの名前タグが赤色になっている
- [ ] 同チームメンバーとの衝突判定がない（すり抜ける）

---

## TC-004: ボット参加とチーム割り当て

**前提条件**
- TC-001 完了済み

**ボット構成**
```bash
node index.js --team red --count 1
```

**手順**
1. 上記コマンドでボットを起動する
2. ゲーム内でボットのチーム所属を確認する
   ```
   /team list
   ```

**期待結果**
- [ ] ボット `Bot_red1` が `kcdk.red` に参加している
- [ ] ボットの名前タグが赤色で表示される
