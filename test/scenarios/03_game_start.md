# 03 ゲーム開始フロー

`/kcdk game start` を実行してから戦闘開始状態になるまでのフローを検証する。

---

## 共通前提条件

- TC-001 完了済み（Scoreboardチーム存在）
- 設定が保存・リロード済み
- ボットまたはプレイヤーが各チームのScoreboardに登録されている

---

## TC-100: 待機地点なし → 即リスポーン地点へTP

**設定**
- red チーム: readyLocation = なし, respawnLocation = (100, 64, 0)
- blue チーム: readyLocation = なし, respawnLocation = (0, 64, 0)

**ボット構成**
```bash
node index.js --team red --count 1 --team blue --count 1
```

**手順**
1. 両チームのメンバーが接続していることを確認
2. ゲームを開始する
   ```
   /kcdk game start
   ```

**期待結果**
- [ ] 赤チームのボットが即座に (100, 64, 0) にTPする
- [ ] 青チームのボットが即座に (0, 64, 0) にTPする
- [ ] TPと同時にゲームモードが設定値（ADVENTURE or SURVIVAL）になる
- [ ] 設定にエフェクトがあれば付与される
- [ ] 設定にアイテムがあれば配布される
- [ ] 設定に革アーマーがあれば着用している

---

## TC-101: 待機地点あり → 待機 → リスポーン地点へTP

**設定**
- red チーム: readyLocation = (50, 64, 0), waitingTime = 30秒, respawnLocation = (100, 64, 0)
- blue チーム: readyLocation = (−50, 64, 0), waitingTime = 10秒, respawnLocation = (0, 64, 0)

**ボット構成**
```bash
node index.js --team red --count 1 --team blue --count 1
```

**手順**
1. ゲームを開始する
   ```
   /kcdk game start
   ```
2. 各チームのTPと動作を観察する

**期待結果（開始直後）**
- [ ] 赤チームが (50, 64, 0) にTPする
- [ ] 青チームが (−50, 64, 0) にTPする
- [ ] 両チームとも **その場から動けない**（移動しようとしても元の位置に戻る）
- [ ] アクションバーに「待機中のチームがいること」が分かる残り時間が表示される

**期待結果（10秒後 — blue の waitingTime 到達）**
- [ ] 青チームが (0, 64, 0) にTPする
- [ ] 青チームが自由に動けるようになる
- [ ] 赤チームはまだ待機中（移動不可）

**期待結果（30秒後 — red の waitingTime 到達）**
- [ ] 赤チームが (100, 64, 0) にTPする
- [ ] 赤チームが自由に動けるようになる
- [ ] アクションバーの待機時間表示が消える

---

## TC-102: チームごとの待機時間が独立して動く

**設定**
- red: waitingTime = 60秒
- blue: waitingTime = 5秒

**手順**
1. ゲームを開始する
2. 5秒後に青チームの動作を確認
3. 60秒後に赤チームの動作を確認

**期待結果**
- [ ] 青チームが5秒後に独立してリスポーン地点へTP
- [ ] 赤チームはその間も待機中（移動不可）
- [ ] 各チームの待機タイマーが共有されていない

**備考**
> バグ記録 #2「待機時間が全チーム共有となっている」の回帰テスト

---

## TC-103: startupCommands の実行

**設定**
```
startupCommands:
  - /say ゲーム開始！
  - /effect give @a minecraft:regeneration 10 1
```

**手順**
1. ゲームを開始する

**期待結果**
- [ ] `ゲーム開始！` のチャットメッセージが全員に表示される
- [ ] 全プレイヤーに Regeneration II が付与される
- [ ] コマンドが順番に実行される

---

## TC-104: エフェクト付与（チームレベル）

**設定**
- red チーム: effects = [SPEED 999999 1 false]

**期待結果**
- [ ] ゲーム開始後（リスポーン地点TP後）、赤チームに Speed II が付与される
- [ ] 青チームにはエフェクトが付与されていない

---

## TC-105: エフェクト付与（ロールレベル・extendsEffects=true）

**設定**
- red チーム: effects = [SPEED 999999 1 false]
- red.captain ロール: effects = [STRENGTH 999999 0 false], extendsEffects = true

**期待結果**
- [ ] captain ロールのプレイヤーに Speed II と Strength I が両方付与される
- [ ] 一般 red メンバーには Speed II のみ付与される

---

## TC-106: エフェクト付与（ロールレベル・extendsEffects=false）

**設定**
- red.captain ロール: effects = [STRENGTH 999999 0 false], extendsEffects = false

**期待結果**
- [ ] captain ロールのプレイヤーに Strength I のみ付与される（Speed は付与されない）

---

## TC-107: アイテム配布（チームアイテムのみ）

**設定**
- red チーム: items = [ダイヤモンドの剣, パン]
- red.captain ロール: items = [], extendsItem = false

**期待結果**
- [ ] captain 以外の red メンバーにダイヤモンドの剣とパンが配布される
- [ ] captain にもチームアイテムが配布される（ロールアイテム未設定のため）

## TC-111: ゲーム開始時にビーコンブロックが設置される

**設定**

**手順**

2. 設定した座標を確認する

**期待結果**
- [ ] 設定した座標にビーコンブロックが設置されている

---

## TC-112: ゲーム中の途中参加（新規ログイン）

**前提条件**
- ゲーム実行中
- 新規プレイヤーが Scoreboard で `kcdk.blue` に事前登録されている

**手順**
1. ゲームを開始する（既存メンバーで）
2. 新規プレイヤーがログインする

**期待結果**
- [ ] Scoreboardチームから所属が自動検出される
- [ ] プレイヤーがリスポーン地点にTPする
- [ ] 装備・エフェクト・アイテムが付与される
- [ ] ゲーム参加者リスト（`gm.players`）に登録される

---

## TC-113: 強制終了コマンド

**手順**
1. ゲーム実行中に強制終了する
   ```
   /kcdk game stop
   ```

**期待結果**
- [ ] 全プレイヤーに `ゲームが強制終了しました。` というタイトルが表示される
- [ ] shutdownCommands が実行される
- [ ] ゲーム状態が停止状態になる

**手順**（カスタムメッセージ）
```
/kcdk game stop §e管理者がゲームを終了しました
```

**期待結果**
- [ ] 指定したメッセージがタイトルで表示される
