# 04 戦闘・リスポーン

FriendlyFire制御、死亡・リスポーン、K/D管理を検証する。

---

## 共通前提条件

- ゲームが RUNNING 状態
- 各テストは `/kcdk game start` から始め、終了後 `/kcdk game stop` でリセット

---

## A. FriendlyFire制御

### TC-200: 同一チーム・同一ロール間のFriendlyFire禁止

**設定**
- red チーム: メンバー2名（ロールなし）

**ボット構成**
```bash
node index.js --team red --count 2
```

**手順**
1. ゲーム開始
2. `Bot_red1` と `Bot_red2` を接近させる
3. 片方が剣で攻撃する

**期待結果**
- [ ] ダメージが入らない（体力が減らない）
- [ ] エラーメッセージ等は表示されない

---

### TC-201: 同一チーム・異なるロール間のFriendlyFire禁止

**設定**
- red チーム: メンバー1名（ロールなし）、captain ロール1名

**ボット構成**
```bash
node index.js --team red --count 1 --team red --captain --count 1
```

**手順**
1. ゲーム開始
2. `Bot_red1`（一般）と `Bot_redCap1`（captain）を接近させて攻撃

**期待結果**
- [ ] ダメージが入らない（同一チームとして扱われる）

**備考**
> バグ記録 #11「ロールが違う場合でもこの判定は同一チームとして扱うこと」の回帰テスト

## B. 死亡・リスポーン

### TC-210: 死亡 → リスポーン地点へTP（respawnCount=-1）

**設定**
- red チーム: respawnCount = -1, respawnLocation = (100, 64, 0)

**手順**
1. ゲーム開始
2. 赤チームプレイヤーをキルする

**期待結果**
- [ ] 死亡後、リスポーン地点 (100, 64, 0) にTPする
- [ ] ゲームモードが維持される（スペクテイターにならない）
- [ ] respawnCount が無限のため何度でもリスポーンできる
- [ ] リスポーン時に装備・アイテム・エフェクトが再付与される

---

### TC-211: respawnCount 消費（respawnCount=2）

**設定**
- red チーム: respawnCount = 2

**手順**
1. ゲーム開始
2. 赤チームプレイヤーを3回キルする

**期待結果**
- [ ] 1回目の死亡: リスポーン（残り1回）
- [ ] 2回目の死亡: リスポーン（残り0回）
- [ ] 3回目の死亡: スペクテイターになる（リスポーン不可）

---

### TC-212: respawnCount=0 → 即スペクテイター

**設定**
- blue チーム: respawnCount = 0

**手順**
1. ゲーム開始
2. 青チームプレイヤーを1回キルする

**期待結果**
- [ ] 死亡後、即スペクテイターモードになる
- [ ] スペクテイターはゲーム上「死亡（戦線離脱）」として扱われる

---

### TC-213: スペクテイターのログアウト→再ログイン

**前提条件**
- ゲーム実行中、対象プレイヤーがすでにスペクテイター状態

**手順**
1. スペクテイターのプレイヤーがサーバーを離れる
2. 同プレイヤーが再接続する

**期待結果**
- [ ] 再接続後もスペクテイターのまま（リスポーンされない）

---

### TC-214: 生存プレイヤーのログアウト→再ログイン（ゲーム中）

**前提条件**
- ゲーム実行中
- プレイヤーが生存状態（残りリスポーンあり）

**手順**
1. 生存プレイヤーがサーバーを離れる
2. 同プレイヤーが再接続する

**期待結果**
- [ ] 再接続後、そのまま生存状態が維持される
- [ ] リスポーン地点への強制TPは行われない（再ログイン時は現位置維持）

## C. K/D管理

### TC-220: キル時のタイトル演出

**手順**
1. ゲーム開始
2. 初めて敵をキルする

**期待結果**
- [ ] キルした側のプレイヤーに以下のタイトルが表示される
  - タイトル: `1 Kill`
  - サブタイトル: キルされたプレイヤーの名前
- [ ] 表示時間が 0.5秒程度である

---

### TC-221: キル数カウントアップ

**手順**
1. 同一ゲーム内で2回目のキルをする

**期待結果**
- [ ] タイトルが `2 Kill` になる

---

2. 赤チームメンバーを3回キルする

**期待結果**
- [ ] チケットが3回消費されゲームが終了する（→ TC-306 参照）

## D. 空腹値制御

### TC-230: disableHunger=true でゲーム中空腹値固定

**設定**
```
/kcdk config disableHunger true
```

**手順**
1. ゲーム開始
2. プレイヤー（またはボット）が走り回り、1分以上経過させる

**期待結果**
- [ ] 空腹ゲージが 20（満タン）のまま変化しない
- [ ] 空腹値が減少しない

---

### TC-231: disableHunger=false でゲーム中空腹値が通常減少

**設定**
```
/kcdk config disableHunger false
```

**手順**
1. SURVIVAL モードでゲーム開始
2. プレイヤーが走り回る

**期待結果**
- [ ] 通常通り空腹ゲージが減少する
