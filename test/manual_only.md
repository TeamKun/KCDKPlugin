# 手動テスト必須シナリオ一覧

Mineflayer では自動化できない、または自動化コストが高すぎて
**現時点では手動確認が必要**なテストケースをまとめる。

自動化できるシナリオは [README.md](README.md) の各シナリオファイルを参照。

---

## 凡例

| 理由コード | 意味 |
|-----------|------|
| `VIS` | 視覚的確認が必要（色・見た目・画面レイアウト） |
| `SND` | 音声確認が必要（SE・BGM） |
| `TIME` | 表示時間・タイミングの目視確認が必要 |
| `EFFECT` | エフェクト付与の確認（metadata パースが複雑） |
| `GUI` | インベントリ GUI の操作・外観確認 |
| `REBOOT` | サーバー再起動を要する確認 |
| `FMT` | アクションバー等の文字フォーマット・色コードの目視確認 |

---

## 1. セットアップ（01_setup.md）

| TC | タイトル | 理由 | 補足 |
|----|---------|------|------|
| TC-003 | プレイヤーの名前タグの色確認 | `VIS` | `kcdk.red` に参加後、名前タグが赤色になるか目視確認。Mineflayer からは他エンティティのチームカラーを直接読み出せない |
| TC-001 | プラグイン起動時チーム自動作成 | `REBOOT` | サーバー再起動を挟んだ確認。ボットは起動前には接続できないため、起動直後の状態確認は手動 |

---

## 2. 設定コマンド（02_config.md）

| TC | タイトル | 理由 | 補足 |
|----|---------|------|------|
| TC-022 | armorColor 設定 | `VIS` | 革アーマーの染色色（HEX）が正しいかは視覚確認が最も確実。NBT 解析でも可能だが実装コストが高い |
| TC-028 | エフェクト追加・削除 | `EFFECT` | 設定コマンド後の `/kcdk config show` の返答パースは可能だが、実際にエフェクトが付与されるかはゲーム中に確認（→ TC-104〜106 へ連携） |
| TC-029 | アイテム設定（GUIコマンド） | `GUI` | インベントリ GUI を開いてアイテムを配置・保存するフローそのものが GUI 操作テスト。自動化は「ウィンドウクリック送信」で技術的に可能だが、GUI タイトル判別・スロット配置の目視確認が必要 |
| TC-037 | ロールアイテム設定（GUIコマンド） | `GUI` | TC-029 と同様 |
| TC-033 | extendsEffects=true の確認 | `EFFECT` | ゲーム開始後に両エフェクトが付与されているかの確認（→ TC-105 と同一内容） |
| TC-034 | extendsEffects=false の確認 | `EFFECT` | 親エフェクトが付与されていないことの確認（→ TC-106 と同一内容） |

## 3. ゲーム開始フロー（03_game_start.md）

| TC | タイトル | 理由 | 補足 |
|----|---------|------|------|
| TC-104 | エフェクト付与（チームレベル） | `EFFECT` | `bot.entity.metadata` の ActiveEffects（index 8）パースが必要で、1.21 のメタデータ仕様に依存して複雑 |
| TC-105 | エフェクト付与（ロール・extendsEffects=true） | `EFFECT` | 2種のエフェクトが両方付いているか確認。metadata パースの他、Speed は移動速度の変化で間接確認できるが不確実 |
| TC-106 | エフェクト付与（ロール・extendsEffects=false） | `EFFECT` | 親エフェクトが付いていないことの確認。欠如の確認はより難しい |
| TC-109 | 革アーマーカラー付与 | `VIS` | チェストプレートの色（HEX）が設定値と一致するか。`bot.entity.equipment[2].nbt` で NBT 解析は可能だが、実際の表示色との照合は目視が確実 |

## 4. 戦闘・リスポーン（04_combat.md）

| TC | タイトル | 理由 | 補足 |
|----|---------|------|------|
| TC-204 | 弓矢命中フィードバック音 | `SND` | 射手に経験値オーブ音が鳴るかの確認。Mineflayer は `named_sound_effect` パケットを受信可能だが、実際に音が鳴るかは人間が聴いて確認するのが確実 |

## 5. 終了条件（05_end_condition.md）

| TC | タイトル | 理由 | 補足 |
|----|---------|------|------|
| TC-308 | composite AND — 片方だけでは終了しない | `IMPORT` | `/kcdk config import` コマンドの応答がボット側でタイムアウトするため自動化困難。composite ロジック自体は extermination（TC-304）・ticket（TC-306）の組み合わせで実装を担保済み |
| TC-309 | composite OR — 片方で終了する | `IMPORT` | TC-308 と同様の理由 |
| TC-310 | ゲーム終了時に shutdownCommands が実行される | `IMPORT` | shutdownCommands は CLI コマンドで設定できず import のみ対応。import 応答のタイムアウト問題が解消されるまで手動確認 |

## 6. UI・表示（06_ui.md）

| TC | タイトル | 理由 | 補足 |
|----|---------|------|------|
| TC-060 | アクションバー表示フォーマット（経過時間） | `FMT` | `bot.on('actionBar')` で文字列は取れるが、色コード（`§e` 等）・区切り記号・フォーマット仕様の確認は目視が必要 |
| TC-061 | アクションバー表示フォーマット（カウントダウン） | `FMT` | 同上。残り時間の表示形式（`1:59` か `01:59` か等）の確認 |
| TC-063 | ボスバーのゲージ変化（ビーコン HP 連動） | `VIS` | `bot.bossBars[0].health` で数値は取れるが、実際の画面上のゲージ長さ・色の変化は目視確認 |
| TC-070 | ゲーム中タブのチーム名表記なし | `VIS` | `bot.tablist` はヘッダー・フッターのみで、個別プレイヤー名の表示フォーマット（プレフィックスが付いているか）は目視確認が確実 |
| TC-074 | Kill 通知タイトルの表示時間（0.5秒） | `TIME` | タイトルテキストは `bot.on('title')` で確認できるが、Stay/Fade の tick 数を設定通りに目視で体感確認する必要がある |
| TC-075 | Kill 通知タイトルの表示時間（n 回目） | `TIME` | TC-074 と同様 |

## 全体を通じた手動確認の前提条件

以下は自動テストを実行する前に必ず手動で設定しておくこと。

| 項目 | 内容 |
|------|------|
| OP 設定 | テスト用ボットの名前（`Bot_red1` 等）を事前にコンソールで OP 登録する: `op Bot_red1` |
| online-mode | `server.properties` の `online-mode=false` を確認する |
| 初回セットアップ | サーバー初回起動後に `/kcdk setup` を手動実行してチームを作成する |
