# 06 UI・表示

アクションバー、ボスバー、タブリスト、タイトル演出を検証する。

---

## 共通前提条件

- ゲームが RUNNING 状態
- 人間プレイヤーが少なくとも1名（UIを目視確認するため）

---

## A. アクションバー

### TC-060: 経過時間の表示（timeLimit=null）

**設定**
```
/kcdk config timeLimit remove
```

**手順**
1. ゲーム開始
2. アクションバーを観察する

**期待結果**
- [ ] `00:05`、`00:10` のように経過時間がリアルタイム表示される
- [ ] 毎tick（1/20秒）更新される（滑らかに変化する）
- [ ] 時間制限のカウントダウンは表示されない

---

### TC-061: 残り時間カウントダウンの表示（timeLimit あり）

**設定**
```
/kcdk config timeLimit set 0 2 0
```

**手順**
1. ゲーム開始
2. アクションバーを観察する

**期待結果**
- [ ] `残り 1:59`、`残り 1:58` … のようにカウントダウンが表示される
- [ ] 毎tick更新で滑らかに変化する

---

### TC-062: チームごとの生存人数表示

**設定**
- red チーム 2名、blue チーム 3名（respawnCount=0）

**ボット構成**
```bash
node index.js --team red --count 2 --team blue --count 3
```

**手順**
1. ゲーム開始
2. アクションバーを確認する
3. blue チームの1名をキルする
4. アクションバーを再確認する

**期待結果**
- [ ] 開始時: `red: 2 / blue: 3` のようなチーム別生存人数が表示される
- [ ] 1名キル後: `red: 2 / blue: 2` に更新される

**備考**
> バグ記録 #4「アクションバーに各チームの生存人数が表示されていない」の回帰テスト

---

### TC-063: 待機中の残り時間表示

**設定**
- red チーム: readyLocation あり, waitingTime = 30秒
- blue チーム: readyLocation なし（即開始）

**手順**
1. ゲーム開始
2. 開始直後のアクションバーを確認する

**期待結果**
- [ ] 待機中チームの残り待機時間がアクションバーに表示される（例: `red待機: 29`）
- [ ] 青チームが待機終了後も赤チームの待機時間が表示され続ける
- [ ] 赤チームの待機時間が0になったら待機時間表示が消える

**備考**
> バグ記録 #10「待機しているチームがいる場合はアクションバーにゲーム開始までの時間を表示すること」の回帰テスト

---

## B. ボスバー

### TC-063: ボスバー表示（bossbar 設定あり）

**設定**
```
/kcdk config bossbar set <HPを持つMobのMCID>
```

**手順**
1. ゲーム開始

**期待結果**
- [ ] ゲーム中、画面上部にボスバーが表示される
- [ ] ビーコンのHPに連動してボスバーのゲージが変化する（beacon終了条件設定時）

---

### TC-064: ボスバー非表示（bossbar=null）

**設定**
```
/kcdk config bossbar remove
```

**手順**
1. ゲーム開始

**期待結果**
- [ ] ボスバーが表示されない

---

## C. タブリスト

### TC-070: ゲーム中のタブ — チーム名表記なし

**手順**
1. ゲーム開始
2. TABキーでタブリストを開く

**期待結果**
- [ ] タブにプレイヤー名のみ表示される
- [ ] チーム名（`[kcdk.red]` など）が付いていない

**備考**
> バグ記録 #6「ゲーム中タブに不要なチーム名表記がある」の回帰テスト

---

### TC-071: ゲーム終了後のタブ — K/D表示

**手順**
1. ゲームを実行する（K/D を発生させる）
2. ゲームを終了させる
3. TABキーでタブリストを開く

**期待結果**
- [ ] プレイヤー名が `<MCID> k:X/d:Y` 形式になっている
  - 例: `Steve k:3/d:1`
- [ ] キル数・デス数が正確である

---

### TC-072: 次のゲーム開始時にK/Dがリセットされる

**手順**
1. TC-071 の状態でそのまま新しいゲームを開始する
2. タブリストを確認する

**期待結果**
- [ ] 開始時点でタブのK/D表示がリセットされている（前ゲームのデータが残らない）

---

## D. タイトル演出（Kill通知）

### TC-074: キル通知タイトル — 1回目

**手順**
1. ゲーム開始
2. 敵を初めてキルする

**期待結果**
- [ ] キルした側のプレイヤー画面に:
  - タイトル: `1 Kill`
  - サブタイトル: キルされたプレイヤーのMCID
- [ ] 表示時間が短い（約0.5秒）
- [ ] 自分以外のプレイヤーには表示されない

**備考**
> バグ記録 #5「敵をキルした時のタイトル演出にキル数が表示されない」の回帰テスト

---

### TC-075: キル通知タイトル — n回目

**手順**
1. 2回目のキルをする

**期待結果**
- [ ] タイトルが `2 Kill` になる
- [ ] 3回目: `3 Kill` ...

---

### TC-076: ゲーム終了タイトル

**手順**
1. 終了条件を達成してゲームを終了させる

**期待結果**
- [ ] 全員の画面に終了メッセージがタイトル表示される
- [ ] メッセージが設定した `endCondition.message` と一致する
- [ ] タイトル表示後、shutdownCommandsが実行される

---

## E. 空腹値表示（disableHunger）

### TC-073: disableHunger=true でゲーム中ハートバー変化なし

**設定**
```
/kcdk config disableHunger true
```
ゲームモード: SURVIVAL

**手順**
1. ゲーム開始
2. プレイヤーが走り続ける（30秒以上）

**期待結果**
- [ ] 空腹ゲージが 10本（満タン）のまま変化しない
- [ ] 空腹によるHP自然回復・ダメージが無効
